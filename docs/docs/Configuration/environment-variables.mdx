Langbuilder uses environment variables to configure certain settings.
You can also import environment variables for use in your deployment, such as environment variables used by certain components in your flows.

You can set Langbuilder environment variables in your terminal, in `.env`, and with the [Langbuilder CLI](./configuration-cli).

## Precedence {#precedence}

If an environment variable is set in multiple places, the following hierarchy applies:

1. Langbuilder CLI options override `.env` and terminal variables.
2. `.env` overrides terminal variables.
3. Terminal variables are used only if the variable isn't set in `.env` or Langbuilder CLI options.

For example, if you set `LANGBUILDER_PORT` in `.env` and your terminal, then Langbuilder uses the value from `.env`.
Similarly, if you run a Langbuilder CLI command with `--port`, Langbuilder uses that port number instead of the `LANGBUILDER_PORT` in `.env`.

## Configure environment variables

Langbuilder recognizes [supported environment variables](#supported-variables) from the following sources:

- Environment variables that you've set in your terminal.
- Environment variables that you've imported from a `.env` file when starting Langbuilder or using the `--env-file` option in the Langbuilder CLI.

You can choose to use one or both sources.
However, environment variables imported from a `.env` file take [precedence](#precedence) over those set in your terminal.

### Set environment variables in your terminal {#configure-variables-terminal}


Run the following commands to set environment variables for your current terminal session:

<Tabs>
<TabItem value="linux-macos" label="Linux or macOS" default>

```bash
export VARIABLE_NAME='VALUE'
```

</TabItem>
<TabItem value="windows" label="Windows">

```
set VARIABLE_NAME='VALUE'
```

</TabItem>
<TabItem value="docker" label="Docker">

```bash
docker run -it --rm \
    -p 7860:7860 \
    -e VARIABLE_NAME='VALUE' \
    nickchasecg/langbuilder:latest
```

</TabItem>
</Tabs>

When you start Langbuilder, it looks for environment variables that you've set in your terminal.
If it detects a supported environment variable, then it automatically adopts the specified value, subject to [precedence rules](#precedence).

### Import environment variables from a .env file {#configure-variables-env-file}

1. If Langbuilder is running, quit Langbuilder.

2. Create a `.env` file, and then open it in your preferred editor.

3. Define [Langbuilder environment variables](#supported-variables) in the `.env` file. For example:

    ```text
    DO_NOT_TRACK=True
    LANGBUILDER_AUTO_LOGIN=False
    LANGBUILDER_AUTO_SAVING=True
    LANGBUILDER_AUTO_SAVING_INTERVAL=1000
    LANGBUILDER_BACKEND_ONLY=False
    LANGBUILDER_BUNDLE_URLS=["https://github.com/user/repo/commit/hash"]
    LANGBUILDER_CACHE_TYPE=async
    LANGBUILDER_COMPONENTS_PATH=/path/to/components/
    LANGBUILDER_CONFIG_DIR=/path/to/config/
    LANGBUILDER_DATABASE_URL=postgresql://user:password@localhost:5432/langbuilder
    LANGBUILDER_DEV=False
    LANGBUILDER_FALLBACK_TO_ENV_VAR=False
    LANGBUILDER_HEALTH_CHECK_MAX_RETRIES=5
    LANGBUILDER_HOST=localhost
    LANGBUILDER_LANGCHAIN_CACHE=InMemoryCache
    LANGBUILDER_MAX_FILE_SIZE_UPLOAD=10000
    LANGBUILDER_MAX_ITEMS_LENGTH=100
    LANGBUILDER_MAX_TEXT_LENGTH=1000
    LANGBUILDER_LOG_LEVEL=error
    LANGBUILDER_OPEN_BROWSER=False
    LANGBUILDER_PORT=7860
    LANGBUILDER_REMOVE_API_KEYS=False
    LANGBUILDER_SAVE_DB_IN_CONFIG_DIR=True
    LANGBUILDER_SECRET_KEY=somesecretkey
    LANGBUILDER_STORE=True
    LANGBUILDER_STORE_ENVIRONMENT_VARIABLES=True
    LANGBUILDER_SUPERUSER=adminuser
    LANGBUILDER_SUPERUSER_PASSWORD=adminpass
    LANGBUILDER_WORKER_TIMEOUT=60000
    LANGBUILDER_WORKERS=3
    ```

   For additional examples, see the [`.env.example`](https://github.com/cloudgeometry/langbuilder/blob/main/.env.example) file in the Langbuilder repository.

4. Save and close `.env`.

5. Start Langbuilder with your `.env` file:

    <Tabs>
    <TabItem value="local" label="Local" default>

    ```bash
    python -m langbuilder run --env-file .env
    ```

    </TabItem>
    <TabItem value="docker" label="Docker">

    ```bash
    docker run -it --rm \
        -p 7860:7860 \
        --env-file .env \
        nickchasecg/langbuilder:latest
    ```

    </TabItem>
    </Tabs>

    If your `.env` file isn't in the same directory, provide the path to your `.env` file.

On startup, Langbuilder imports the environment variables from your `.env` file, as well as any others that you set in your terminal, and then adopts their specified values.

## Supported environment variables {#supported-variables}

The following table lists the environment variables supported by Langbuilder.

| Variable | Format | Default | Description |
|----------|--------|---------|-------------|
| `DO_NOT_TRACK` | Boolean | False | Whether to report [Langbuilder telemetry data](/contributing-telemetry). If true, Langbuilder telemetry is disabled. |
| `LANGBUILDER_AUTO_LOGIN` | Boolean | True | See [`LANGBUILDER_AUTO_LOGIN`](/api-keys-and-authentication#langbuilder-auto-login). |
| `LANGBUILDER_AUTO_SAVING` | Boolean | True | Enable flow auto-saving. |
| `LANGBUILDER_AUTO_SAVING_INTERVAL` | Integer | `1000` | Set the interval for flow auto-saving in milliseconds. |
| `LANGBUILDER_BACKEND_ONLY` | Boolean | False | Run only the Langbuilder backend service (no frontend). |
| `LANGBUILDER_BUNDLE_URLS` | List[String] | `[]` | A list of URLs from which to load component bundles and flows. Supports GitHub URLs. If LANGBUILDER_AUTO_LOGIN is enabled, flows from these bundles are loaded into the database. |
| `LANGBUILDER_CACHE_TYPE` | String | `async` | Set the cache type for Langbuilder. Possible values: `async`, `redis`, `memory`, `disk`. If you set the type to `redis`, then you must also set the following environment variables: `LANGBUILDER_REDIS_HOST`, `LANGBUILDER_REDIS_PORT`, `LANGBUILDER_REDIS_DB`, and `LANGBUILDER_REDIS_CACHE_EXPIRE`. |
| `LANGBUILDER_COMPONENTS_PATH` | String | Not set | Path to the directory containing custom components. |
| `LANGBUILDER_CONFIG_DIR` | String | Varies | Set the Langbuilder configuration directory where files, logs, and the Langbuilder database are stored. Default path depends on your installation. See [Flow storage and logs](/concepts-flows#flow-storage-and-logs). |
| `LANGBUILDER_DATABASE_URL` | String | Not set | Set the database URL for Langbuilder. If not provided, Langbuilder uses a SQLite database. |
| `LANGBUILDER_DATABASE_CONNECTION_RETRY` | Boolean | False | Whether to retry lost connections to the Langbuilder database. If true, Langbuilder tries to connect to the database again if the connection fails. |
| `LANGBUILDER_DB_POOL_SIZE` | Integer | `20` | **DEPRECATED:** Use `LANGBUILDER_DB_CONNECTION_SETTINGS` instead. The number of connections to keep open in the connection pool. |
| `LANGBUILDER_DB_MAX_OVERFLOW` | Integer | `30` | **DEPRECATED:** Use `LANGBUILDER_DB_CONNECTION_SETTINGS` instead. The number of connections to allow that can be opened beyond the pool size. |
| `LANGBUILDER_DB_CONNECT_TIMEOUT` | Integer | `30` | The number of seconds to wait before giving up on a lock to be released or establishing a connection to the database. |
| `LANGBUILDER_DB_CONNECTION_SETTINGS` | JSON | Not set | A JSON dictionary to centralize database connection parameters. Example: `{"pool_size": 20, "max_overflow": 30}` |
| `LANGBUILDER_DISABLE_TRACK_APIKEY_USAGE` | Boolean | False | Whether to track API key usage. If true, disables tracking of API key usage (`total_uses` and `last_used_at`) to avoid database contention under high concurrency. |
| `LANGBUILDER_ENABLE_SUPERUSER_CLI` | Boolean | True | Allow creation of superusers with the Langbuilder CLI command [`langbuilder superuser`](./configuration-cli.mdx#langbuilder-superuser). Recommended to be disabled (false) in production for security reasons. |
| `LANGBUILDER_FALLBACK_TO_ENV_VAR` | Boolean | True | If enabled, [global variables](/configuration-global-variables) set in your Langbuilder **Settings** can use an environment variable with the same name if Langbuilder can't retrieve the variable value from the global variables. |
| `LANGBUILDER_FRONTEND_PATH` | String | `./frontend` | Path to the frontend directory containing build files. This is for development purposes only. See [`--frontend-path`](./configuration-cli.mdx#run-frontend-path). |
| `LANGBUILDER_HEALTH_CHECK_MAX_RETRIES` | Integer | `5` | Set the maximum number of retries for the health check. See [`--health-check-max-retries`](./configuration-cli.mdx#run-health-check-max-retries). |
| `LANGBUILDER_HOST` | String | `localhost` | The host on which the Langbuilder server will run. See [`--host`](./configuration-cli.mdx#run-host). |
| `LANGBUILDER_LANGCHAIN_CACHE` | String | `InMemoryCache` | Type of cache to use. Possible values: `InMemoryCache`, `SQLiteCache`. See [`--cache`](./configuration-cli.mdx#run-cache). |
| `LANGBUILDER_LOG_LEVEL` | String | `INFO` | Set the logging level for Langbuilder. Possible values: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`. |
| `LANGBUILDER_LOG_FILE` | String | Not set | Path to the log file. If this option isn't set, logs are written to stdout. |
| `LANGBUILDER_LOG_RETRIEVER_BUFFER_SIZE` | Integer | `10000` | Set the buffer size for log retrieval. Only used if `LANGBUILDER_ENABLE_LOG_RETRIEVAL` is enabled. |
| `LANGBUILDER_MAX_FILE_SIZE_UPLOAD` | Integer | `100` | Set the maximum file size for the upload in megabytes. See [`--max-file-size-upload`](./configuration-cli.mdx#run-max-file-size-upload). |
| `LANGBUILDER_MAX_ITEMS_LENGTH` | Integer | `100` | Maximum number of items to store and display in the visual editor. Lists longer than this will be truncated when displayed in the visual editor. Doesn't affect data passed between components nor outputs. |
| `LANGBUILDER_MAX_TEXT_LENGTH` | Integer | `1000` | Maximum number of characters to store and display in the visual editor. Responses longer than this will be truncated when displayed in the visual editor. Doesn't truncate responses between components nor outputs. |
| `LANGBUILDER_MCP_SERVER_ENABLED` | Boolean | True | If this option is set to False, Langbuilder doesn't enable the MCP server. |
| `LANGBUILDER_MCP_SERVER_ENABLE_PROGRESS_NOTIFICATIONS` | Boolean | False | If this option is set to True, Langbuilder sends progress notifications in the MCP server. |
| `LANGBUILDER_NEW_USER_IS_ACTIVE` | Boolean | False | See [`LANGBUILDER_NEW_USER_IS_ACTIVE`](/api-keys-and-authentication#langbuilder-new-user-is-active). |
| `LANGBUILDER_OPEN_BROWSER` | Boolean | False | Open the system web browser on startup. |
| `LANGBUILDER_PORT` | Integer | `7860` | The port on which the Langbuilder server runs. The server automatically selects a free port if the specified port is in use. |
| `LANGBUILDER_PROMETHEUS_ENABLED` | Boolean | False | Expose Prometheus metrics. |
| `LANGBUILDER_PROMETHEUS_PORT` | Integer | `9090` | Set the port on which Langbuilder exposes Prometheus metrics. |
| `LANGBUILDER_REDIS_CACHE_EXPIRE` | Integer | `3600` | See `LANGBUILDER_CACHE_TYPE`. |
| `LANGBUILDER_REDIS_DB` | Integer | `0` | See `LANGBUILDER_CACHE_TYPE`. |
| `LANGBUILDER_REDIS_HOST` | String | `localhost` | See `LANGBUILDER_CACHE_TYPE`. |
| `LANGBUILDER_REDIS_PORT` | String | `6379` | See `LANGBUILDER_CACHE_TYPE`. |
| `LANGBUILDER_REDIS_PASSWORD` | String | Not set | Password for Redis authentication when using Redis cache type. |
| `LANGBUILDER_REMOVE_API_KEYS` | Boolean | False | Remove API keys from the projects saved in the database. |
| `LANGBUILDER_SAVE_DB_IN_CONFIG_DIR` | Boolean | False | If false (default), the Langbuilder database is saved in the `langbuilder` root directory. This means the database isn't shared between different virtual environments, and the database is deleted when you uninstall Langbuilder. If true, the database is saved in the `LANGBUILDER_CONFIG_DIR`. |
| `LANGBUILDER_SECRET_KEY` | String | Automated | See [`LANGBUILDER_SECRET_KEY`](/api-keys-and-authentication#langbuilder-secret-key). |
| `LANGBUILDER_STORE` | Boolean | True | Whether to enable the Langbuilder Store features. |
| `LANGBUILDER_STORE_ENVIRONMENT_VARIABLES` | Boolean | True | Whether to store environment variables as [global variables](/configuration-global-variables) in the database. |
| `LANGBUILDER_CREATE_STARTER_PROJECTS` | Boolean | True | Whether to create templates during initialization. If false, Langbuilder doesn't create templates, and `LANGBUILDER_UPDATE_STARTER_PROJECTS` is treated as false. |
| `LANGBUILDER_UPDATE_STARTER_PROJECTS` | Boolean | True | Whether to update templates with the latest component versions when initializing after an upgrade. |
| `LANGBUILDER_SUPERUSER` | String | `langbuilder` | See [`LANGBUILDER_SUPERUSER` and `LANGBUILDER_SUPERUSER_PASSWORD`](/api-keys-and-authentication#langbuilder-superuser). |
| `LANGBUILDER_SUPERUSER_PASSWORD` | String | `langbuilder` | See [`LANGBUILDER_SUPERUSER` and `LANGBUILDER_SUPERUSER_PASSWORD`](/api-keys-and-authentication#langbuilder-superuser). |
| `LANGBUILDER_VARIABLES_TO_GET_FROM_ENVIRONMENT` | String | Not set | Comma-separated list of environment variables to get from the environment and store as [global variables](/configuration-global-variables). |
| `LANGBUILDER_LOAD_FLOWS_PATH` | String | Not set | Path to a directory containing flow JSON files to be loaded on startup. Note that this feature only works if `LANGBUILDER_AUTO_LOGIN` is enabled. |
| `LANGBUILDER_WORKER_TIMEOUT` | Integer | `300` | Worker timeout in seconds. |
| `LANGBUILDER_WORKERS` | Integer | `1` | Number of worker processes. |
| `LANGBUILDER_SSL_CERT_FILE` | String | Not set | Path to the SSL certificate file on the local system. |
| `LANGBUILDER_SSL_KEY_FILE` | String | Not set | Path to the SSL key file on the local system. |
| `LANGBUILDER_SKIP_AUTH_AUTO_LOGIN` | Boolean | True | See [`LANGBUILDER_AUTO_LOGIN`](/api-keys-and-authentication#langbuilder-auto-login). |

## Set environment variables for Langbuilder Desktop

Environment variables set in your terminal aren't automatically available to GUI-based applications like Langbuilder Desktop when you launch them from the Windows or macOS GUI.

For Windows, this means any GUI-based app launched from the Start menu, desktop shortcuts, or Windows Explorer.

For macOS, this means any GUI-based app launched from Finder, Spotlight, Launchpad, or the Dock.

To set environment variables for Langbuilder Desktop, you need to use specific commands or files, depending on your OS.

<Tabs>
<TabItem value="macos" label="macOS" default>

Langbuilder Desktop for macOS cannot automatically use variables set in your terminal, such as those in`.zshrc` or `.bash_profile`, when launched from the macOS GUI.

To make environment variables available to GUI apps on macOS, you need to use `launchctl` with a `plist` file:

1. Create the `LaunchAgents` directory if it doesn't exist:

    ```bash
    mkdir -p ~/Library/LaunchAgents
    ```

2. In the `LaunchAgents` directory, create a `.plist` file called `dev.langbuilder.env`.

3. Add the following content to `dev.langbuilder.env.plist`, and then add, change, or remove Langbuilder environment variables as needed for your configuration.

    This example sets multiple environmental variables for all GUI apps launched from the macOS GUI.

    ```xml
     <?xml version="1.0" encoding="UTF-8"?>
     <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
     <plist version="1.0">
     <dict>
       <key>Label</key>
       <string>dev.langbuilder.env</string>
       <key>ProgramArguments</key>
       <array>
         <string>/bin/sh</string>
         <string>-c</string>
          <string>
            launchctl setenv LANGBUILDER_CONFIG_DIR /Users/your_user/custom/config &&
            launchctl setenv LANGBUILDER_PORT 7860 &&
            launchctl setenv LANGBUILDER_HOST localhost &&
            launchctl setenv ARIZE_API_KEY ak-...
          </string>
       </array>
       <key>RunAtLoad</key>
       <true/>
     </dict>
     </plist>
    ```
4. Load the file with `launchctl`:

    ```bash
    launchctl load ~/Library/LaunchAgents/dev.langbuilder.env.plist
    ```

</TabItem>
<TabItem value="windows1" label="Window System Properties">

Langbuilder Desktop for Windows cannot automatically use variables set in your terminal, such as those defined with `set` in `cmd` or `$env:VAR=...` in PowerShell, when launched from the Windows GUI.

To make environment variables available to the Langbuilder Desktop app, you must set them at the user or system level using the **System Properties** interface or the Terminal.

To set environment variables using the System Properties interface, do the following:

1. Press <kbd>Win + R</kbd>, enter `SystemPropertiesAdvanced`, and then press <kbd>Enter</kbd>.
2. Click **Environment Variables**.
3. Under **User variables**, click **New**.

    :::tip
    To apply the setting to all users, select **System variables**.
    :::

4. Enter the name of the Langbuilder variable you want to set, such as `LANGBUILDER_CONFIG_DIR`, and the desired value, such as `C:\Users\your_user\.langbuilder_config`.
5. Click **OK** to save the variable.
6. Repeat until you have set all necessary Langbuilder environment variables.
7. Launch or restart Langbuilder Desktop to apply the environment variables.

</TabItem>
<TabItem value="windows2" label="Powershell">

To define environment variables for Windows using PowerShell, do the following:

1. Enter the name of the Langbuilder variable you want to set, such as `LANGBUILDER_CONFIG_DIR`, and the desired value, such as `C:\Users\your_user\.langbuilder_config`.

    To set an environment variable for the current user:
    ```powershell
    [System.Environment]::SetEnvironmentVariable("LANGBUILDER_CONFIG_DIR", "C:\Users\your_user\.langbuilder_config", "User")
    ```
